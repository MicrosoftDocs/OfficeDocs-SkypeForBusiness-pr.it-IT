# <a name="shifts-for-teams"></a>Turni per Teams

Teams offre agli operatori I FIRSTLINE dell'organizzazione gli strumenti necessari per comunicare e collaborare efficacemente e svolgere il loro lavoro migliore. Qui troverai le indicazioni per gli amministratori che ti servono per configurare e gestire i turni, lo strumento di Gestione pianificazione, in teams.

### <a name="shifts-extensions"></a>Estensioni di turni

|               |               |
| ------------- | ------------- |
| ![API](../media/api-small.svg) | **[API di Shift Graph](/graph/api/resources/shift?view=graph-rest-1.0)** Le API shifts Graph permettono di integrare i dati di turni con i sistemi di gestione della manodopera esterna, offrendoti la flessibilità necessaria per creare esperienze personalizzate di turni nel back-end, offrendo agli utenti un'esperienza di front-end completa in teams.             |
| ![API](../media/api-small.svg) | **[Integrazioni della gestione della forza lavoro](https://github.com/OfficeDev/Microsoft-Teams-Shifts-WFM-Connectors)** Se si usano sistemi di gestione della forza lavoro di terze parti, ad esempio Kronos e JDA, per la pianificazione, l'ora e la presenza, è possibile integrare direttamente i turni attraverso le API del grafico turni e l'SDK con le integrazioni open source. |
| ![API](../media/process-flow-teams-small.svg) | **[Turni + automatizzazione della potenza](https://github.com/OfficeDev/Microsoft-Teams-Shifts-Power-Automate-Templates)** Turni + Power Automate consente di ottenere informazioni da turni e creare flussi di lavoro personalizzati con altre app ed eseguire operazioni su scala. Automatizzare i processi chiave con codice poco o niente. I trigger e i modelli supportano diversi scenari, ad esempio l'abilitazione dell'approvazione automatica per le richieste di spostamento quando l'approvazione di un Manager non è necessaria. |
