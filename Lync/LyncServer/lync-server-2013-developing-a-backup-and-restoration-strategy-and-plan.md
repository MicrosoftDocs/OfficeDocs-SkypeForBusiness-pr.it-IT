# <a name="developing-a-backup-and-restoration-strategy-and-plan-for-lync-server-2013"></a>Sviluppo di una strategia di backup e ripristino e pianificazione di Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Ultimo argomento modificato:** 2013-02-17_

L'efficacia delle operazioni di backup e ripristino di Lync Server dipende dalla strategia e dal piano di backup e ripristino. È consigliabile definire una strategia per il backup e il ripristino di Lync Server che si adatta alla strategia complessiva dell'organizzazione e un piano completo e conciso per il backup dei dati e delle impostazioni e, in caso di un'interruzione, un piano per il ripristino del servizio.

Per il ripristino di emergenza più affidabile di un pool Front End, utilizzare la topologia di ripristino di emergenza con pool di coppie introdotta in Lync Server 2013. Per ulteriori informazioni, vedere [pianificazione per la disponibilità elevata e il ripristino di emergenza in Lync Server 2013](lync-server-2013-planning-for-high-availability-and-disaster-recovery.md).
