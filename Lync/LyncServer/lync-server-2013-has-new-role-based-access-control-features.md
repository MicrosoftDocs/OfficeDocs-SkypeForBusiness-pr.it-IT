# <a name="new-role-based-access-control-features-in-lync-server-2013"></a>Nuove funzionalità di controllo di accesso basato sui ruoli in Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Argomento Ultima modifica:** 2012-09-20_

Lync Server 2013 migliora la funzionalità di controllo di accesso basato sui ruoli in due modi principali. È ora possibile creare ruoli personalizzati, ognuno dei quali ha privilegi per solo un set di cmdlet specificati. A questi ruoli personalizzati può essere assegnato anche il privilegio di eseguire script di cmdlet.

Inoltre, Lync Server 2013 include due nuovi ruoli predefiniti.

  - Gli utenti che hanno ricevuto il ruolo *Response Group Manager* possono gestire code di Response Group specifiche nell'organizzazione, ma non necessariamente avere diritti di gestione per altre code o l'applicazione Response Group nel suo complesso.

  - Gli utenti che hanno il ruolo di *Manager della chat persistente* possono gestire le chat room permanenti specifiche dell'organizzazione, ma non necessariamente avere diritti di gestione per le altre sale o la caratteristica della chat persistente nel suo complesso.
