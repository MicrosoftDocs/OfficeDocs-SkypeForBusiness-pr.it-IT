# <a name="new-call-park-application-features-in-lync-server-2013"></a>Nuove funzionalità dell'applicazione Parcheggio di chiamata in Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Ultimo argomento modificato:** 2012-10-17_

L'applicazione Parcheggio di chiamata consente agli utenti di VoIP aziendale di inserire una chiamata in attesa e quindi di recuperarla in un secondo momento da qualsiasi telefono. L'utente che ha parcheggiato la chiamata può comporre il numero di orbita fornito dal parcheggio di chiamata per recuperare la chiamata parcheggiata o utilizzare un meccanismo esterno, ad esempio messaggistica istantanea o un sistema di paging, per chiedere a qualcun altro di recuperare la chiamata.

Lync Server 2013 fornisce nuovi meccanismi di ripristino di emergenza in forma di processi di failover e failback. Questi processi di failover e failback supportano il ripristino della funzionalità del parcheggio di chiamata, consentendo agli utenti che si trovano nel pool primario di sfruttare l'applicazione Parcheggio di chiamata del pool di backup quando si verifica un'interruzione nel pool primario. Il supporto per il ripristino di emergenza dell'applicazione Parcheggio di chiamata è abilitato nell'ambito della configurazione e della distribuzione di pool Front End associati.
