# <a name="requirements-for-running-best-practices-analyzer-in-lync-server-2013"></a>Requisiti per l'uso di Best Practices Analyzer in Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Argomento Ultima modifica:** 2012-09-19_

È possibile usare Lync Server 2013, Best Practices Analyzer per analizzare l'ambiente Lync Server 2013. Non è possibile usarlo per analizzare gli ambienti precedenti, ma è possibile usare le versioni precedenti dello strumento per analizzare questi ambienti. Per informazioni dettagliate sul download e l'uso delle versioni di Best Practices Analyzer di Lync Server 2010 e Office Communications Server 2007 R2, vedere "Lync Server 2010, Best [http://go.microsoft.com/fwlink/p/?linkId=210536](http://go.microsoft.com/fwlink/p/?linkid=256358) Practices Analyzer" at e "Best Practices Analyzer per Office communications Server 2007 e Office [http://go.microsoft.com/fwlink/p/?linkId=256358](http://go.microsoft.com/fwlink/p/?linkid=210651)Communications Server 2007 R2".

Prima di avviare la scansione, è necessario assicurarsi che tutti i componenti dell'ambiente Lync Server 2013 siano in uso e online.

<div>


> [!NOTE]  
> A seconda della configurazione degli Edge Server e delle impostazioni di rete perimetrale correlate, incluse le impostazioni e le autorizzazioni del firewall, l'Analizzatore procedure consigliate potrebbe non essere in grado di accedere e analizzare i server perimetrali. Se si includono gli Edge Server nella scansione e i report indicano che si è verificato un problema con l'accesso a Edge Server, è consigliabile rimuovere i server perimetrali dalle opzioni di analisi ed eseguire di nuovo l'analisi in modo che i problemi non vengano visualizzati nel report.
