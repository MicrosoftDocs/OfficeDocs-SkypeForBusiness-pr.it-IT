# <a name="documentation-in-lync-server-2013"></a>Documentazione in Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Argomento Ultima modifica:** 2015-05-15_

Il modello MOF è costituito da numerose funzioni di gestione dei servizi. La documentazione su come e quando eseguire le attività può essere condivisa con i membri dello stesso team o con altri team. Il metodo di archiviazione e condivisione della documentazione può variare in base al tipo di funzione. Le procedure per l'amministrazione di sistema, ad esempio, possono essere archiviate come documenti di Word, in quanto probabilmente verranno stampate e riportate a un riferimento frequente. Le informazioni di gestione della configurazione possono essere generate e archiviate automaticamente in un database per facilitare la ricerca e l'indicizzazione. Una certa documentazione può essere sensibile e deve essere limitata.

<div>

## <a name="document-management-systems"></a>Sistemi di gestione dei documenti

Un sistema di gestione della documentazione funge da repository centrale per i documenti e consente di verificare che sia disponibile solo l'ultima versione di un documento. È anche possibile considerare l'archiviazione della versione precedente del documento come riferimento. Lync Server offre funzionalità adatte per questa attività.

</div>

<div>

## <a name="databases"></a>Database

Sono stati discussi diversi strumenti e funzioni di gestione adatti all'uso dei database. Il processo di gestione della configurazione rischia di usare processi automatizzati che memorizzano grandi quantità di dati che richiedono l'indicizzazione e la ricerca. Il personale di supporto può cercare un database di problemi e risoluzioni precedenti quando si risolvono nuovi problemi.

È probabile che vengano usati database diversi per scopi diversi. Decidere se questi database devono essere collegati o combinati. Ad esempio, se il servizio di assistenza identifica diversi problemi con un tema comune (ad esempio il nuovo software che causa un problema con una specifica scheda di rete), il personale di supporto può eseguire una query nel database di configurazione per prevedere il numero di computer che potrebbero essere interessati.
