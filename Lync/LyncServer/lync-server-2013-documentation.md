# <a name="documentation-in-lync-server-2013"></a>Documentazione in Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Ultimo argomento modificato:** 2015-05-15_

Il modello MOF è costituito da numerose funzioni di gestione dei servizi. La documentazione su come e quando eseguire le attività può essere condivisa con i membri dello stesso team o con altri team. Il metodo di archiviazione e condivisione della documentazione può variare in base al tipo di funzione. Ad esempio, le procedure per l'amministrazione del sistema possono essere archiviate come documenti di Word perché è probabile che vengano stampate e referenziate frequentemente. Le informazioni di gestione della configurazione possono essere generate e archiviate automaticamente in un database per semplificare la ricerca e l'indicizzazione. Alcuni documenti possono essere sensibili e devono essere limitati.

<div>

## <a name="document-management-systems"></a>Sistemi di gestione dei documenti

Un sistema di gestione della documentazione funge da archivio centrale per i documenti e assicura che sia disponibile solo l'ultima revisione di un documento. È inoltre possibile valutare la possibilità di archiviare la versione precedente del documento come riferimento. Lync Server fornisce la funzionalità adatta a questa attività.

</div>

<div>

## <a name="databases"></a>Database

Sono stati descritti diversi strumenti e funzioni di gestione che sono adatti all'utilizzo dei database. È probabile che il processo di gestione della configurazione utilizzi processi automatici che archiviano grandi quantità di dati che richiedono l'indicizzazione e la ricerca. Il personale di supporto può eseguire ricerche in un database di problemi e risoluzioni precedenti durante la risoluzione dei problemi.

È probabile che vengano utilizzati database diversi per scopi diversi. Decidere se questi database devono essere collegati o combinati. Ad esempio, se il service desk identifica diversi problemi con un tema comune (ad esempio un nuovo software che causa un problema con una scheda di rete specifica), il personale di supporto può eseguire query sul database di configurazione per prevedere il numero di computer che potrebbero essere intaccati.
